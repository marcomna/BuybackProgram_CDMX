A continuación, se presenta la descripción de las carpetas que conforman cada paquete de datos abiertos:

Carpeta de catalogos: 
Esta carpeta contiene los catálogos que se relacionan y detallan a los conjuntos de datos, permitiendo así obtener la descripción que representa cada código. Cabe señalar que el nombre de cada catálogo corresponde al nombre de la variable o columna con la que se relaciona dentro de un conjunto de datos.

Carpeta de conjunto_de_datos:
Esta carpeta contiene los conjuntos de datos, es decir, la información captada o que es proporcionada por el informante según su nivel y ámbito de gobierno, en su mayoría, esta información esta codificada. Así también esta carpeta contiene un archivo llamado “Índice” que detalla o describe la información y/o pregunta que contiene cada archivo de conjunto de dato.

Carpeta de diccionario_de_datos:
Esta carpeta contiene lo que conocemos como File Descripción (FD) o diccionario de datos, el cual apoya en la interpretación de la estructura de cada conjunto de datos, además de proporcionar un panorama para las relaciones que existen entre los conjuntos de datos y los catálogos. En este archivo podremos ubicar detalladamente las columnas que se relacionan a un catálogo o si es campo abierto, su tipo de dato y descripción de cada variable o columna. Así también esta carpeta contiene un archivo llamado readme.txt que nos retroalimenta sobre algunos caracteres (siglas) que se utilizan para el manejo de valores que indican o representan algún estatus dentro de la información de los conjuntos de datos.

Carpeta de metadatos:
Esta carpeta contiene un archivo .txt que proporciona información general del programa o proyecto.

Carpeta de modelo_entidad_relación:
Esta carpeta contiene los modelos Entidad – Relación (E-R) que son la base del cómo los conjuntos de datos y los catálogos interactúan entre sí, además de ser la pauta para un probable montaje de todos los conjuntos de datos en un esquema de bases de datos.
